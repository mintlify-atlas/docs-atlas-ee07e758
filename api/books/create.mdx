---
title: "Create Book"
api: "POST /api/v1/books/create"
description: "Creates a new book in the library system."
---

## Authentication

This endpoint requires a valid JWT token in the Authorization header.

```bash
Authorization: Bearer <your_jwt_token>
```

## Request Body

<ParamField body="gender" type="string" required>
  The genre or category of the book (e.g., "Fantasy", "Science Fiction", "Mystery").
</ParamField>

<ParamField body="author" type="string" required>
  The author's name.
</ParamField>

<ParamField body="image" type="string" required>
  URL or path to the book's cover image.
</ParamField>

<ParamField body="title" type="string" required>
  The main title of the book.
</ParamField>

<ParamField body="subtitle" type="string" optional>
  The subtitle of the book.
</ParamField>

<ParamField body="publisher" type="string" required>
  The name of the publisher.
</ParamField>

<ParamField body="year" type="string" required>
  The publication year of the book.
</ParamField>

<ParamField body="pages" type="integer" required>
  The number of pages in the book.
</ParamField>

<ParamField body="isbn" type="string" required>
  The International Standard Book Number (ISBN). Must be unique in the system.
</ParamField>

## Status Codes

<ResponseField name="201" type="Created">
  Book created successfully.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request body or validation errors (e.g., missing required fields, duplicate ISBN).
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid JWT token.
</ResponseField>

<ResponseField name="403" type="Forbidden">
  Valid token but insufficient permissions to create books.
</ResponseField>
