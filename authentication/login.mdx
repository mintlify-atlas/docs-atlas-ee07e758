## Overview

The login endpoint authenticates users and returns a JWT token that must be included in subsequent requests to protected endpoints. The authentication process validates credentials against BCrypt-hashed passwords stored in the database.

## Login Endpoint

<CodeGroup>
```bash cURL
curl -X POST "https://api.example.com/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "johndoe",
    "password": "SecurePass123!"
  }'
```

```javascript JavaScript/Axios
const axios = require('axios');

const loginData = {
  username: 'johndoe',
  password: 'SecurePass123!'
};

axios.post('https://api.example.com/api/v1/auth/login', loginData)
  .then(response => {
    console.log('Login successful:', response.data);
    // Store the JWT token
    const token = response.data.jwt;
    localStorage.setItem('authToken', token);
    console.log('Token stored for future requests');
  })
  .catch(error => {
    if (error.response && error.response.status === 401) {
      console.error('Invalid credentials');
    } else {
      console.error('Login failed:', error.message);
    }
  });
```

```python Python/Requests
import requests

url = 'https://api.example.com/api/v1/auth/login'

data = {
    'username': 'johndoe',
    'password': 'SecurePass123!'
}

response = requests.post(url, json=data)

if response.status_code == 201:
    result = response.json()
    token = result['jwt']
    print('Login successful')
    print(f'Token: {token}')
    # Store token for future use
else:
    print('Login failed:', response.json())
```

```java Java
HttpClient client = HttpClient.newHttpClient();

String json = """{
    "username": "johndoe",
    "password": "SecurePass123!"
}""";

HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://api.example.com/api/v1/auth/login"))
        .header("Content-Type", "application/json")
        .POST(HttpRequest.BodyPublishers.ofString(json))
        .build();

HttpResponse<String> response = client.send(request, 
    HttpResponse.BodyHandlers.ofString());

if (response.statusCode() == 201) {
    System.out.println("Login successful: " + response.body());
} else {
    System.out.println("Login failed: " + response.body());
}
```
</CodeGroup>

## Request Format

### Endpoint Details

- **URL**: `/api/v1/auth/login`
- **Method**: `POST`
- **Content-Type**: `application/json`
- **Authentication**: Not required (public endpoint)

### Request Body Schema

The request body must conform to the `AuthLoginRequest` DTO:

<CodeGroup>
```json Request Body
{
  "username": "string",
  "password": "string"
}
```

```java AuthLoginRequest.java
public record AuthLoginRequest(
    @NotBlank(message = "The username is obligatory") 
    String username,
    
    @NotBlank(message = "The password is obligatory") 
    String password
) {}
```
</CodeGroup>

### Field Requirements

| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| `username` | String | Yes | Not blank | The username associated with the account |
| `password` | String | Yes | Not blank | The password for authentication |

<Info>
  Both fields are mandatory. The API will return a validation error if either field is missing or empty.
</Info>

```java AuthLoginResponse.java
@JsonPropertyOrder({"username", "message", "status", "jwt"})
public record AuthLoginResponse(
    String username,
    String message,
    String jwt,
    Boolean status
) {}
```
</CodeGroup>

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `username` | String | The authenticated username |
| `message` | String | Success message ("User logged in correctly") |
| `jwt` | String | JWT access token for authenticated requests |
| `status` | Boolean | Always `true` for successful login |

<Warning>
  The JWT token is the most critical part of the response. Store it securely and include it in all subsequent API requests to protected endpoints.
</Warning>

## Authentication Process

The login process follows these steps:

<Steps>
  <Step title="Validate Request">
    The API validates that both username and password are provided and not blank
  </Step>
  
  <Step title="Load User Details">
    The system retrieves the user from the database using the provided username
    
    ```java
    AuthUser user = authUserRepository.findAuthUserByUsername(username)
        .orElseThrow(() -> new UsernameNotFoundException(
            "The user " + username + " doesn't exist"));
    ```
  </Step>
  
  <Step title="Verify Password">
    BCrypt is used to compare the provided password with the stored hash
    
    ```java
    if (!passwordEncoder.matches(password, userDetails.getPassword())) {
        throw new BadCredentialsException("Invalid password");
    }
    ```
  </Step>
  
  <Step title="Create Authentication Object">
    A Spring Security Authentication object is created with user details
    
    ```java
    Authentication authentication = new UsernamePasswordAuthenticationToken(
        userDetails.getUsername(),
        userDetails.getPassword(),
        userDetails.getAuthorities()
    );
    ```
  </Step>
  
  <Step title="Generate JWT Token">
    A JWT token is created using the JwtUtils utility
    
    ```java
    String accessToken = jwtUtils.createToken(authentication);
    ```
  </Step>
  
  <Step title="Return Response">
    The API returns the username, success message, JWT token, and status
    
    ```java
    return new AuthLoginResponse(
        username, 
        "User logged in correctly", 
        accessToken, 
        true
    );
    ```
  </Step>
</Steps>

## JWT Token Details

The JWT token returned upon successful login contains the following information:

### Token Structure

```
header.payload.signature
```

### Token Claims

| Claim | Description | Example |
|-------|-------------|----------|
| `iss` | Issuer of the token | Configured in `security.jwt.user.generator` |
| `sub` | Subject (username) | `"johndoe"` |
| `authorities` | User authorities/roles | `""` (empty in current implementation) |
| `iat` | Issued at timestamp | `1709640000` |
| `exp` | Expiration timestamp | `1709641800` (30 minutes after `iat`) |
| `jti` | Unique token ID | `"12345678-1234-1234-1234-1234567890ab"` |
| `nbf` | Not before timestamp | `1709640000` |

<Info>
  JWT tokens expire **30 minutes** (1,800,000 milliseconds) after issuance. You must re-authenticate to obtain a new token after expiration.
</Info>

## Using the JWT Token

After successful login, include the JWT token in the `Authorization` header of all requests to protected endpoints:

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/api/v1/books" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```javascript JavaScript/Axios
const token = localStorage.getItem('authToken');

axios.get('https://api.example.com/api/v1/books', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
})
.then(response => {
  console.log('Books:', response.data);
})
.catch(error => {
  if (error.response && error.response.status === 401) {
    console.error('Token expired or invalid');
    // Redirect to login
  }
});
```

```python Python/Requests
import requests

token = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'

headers = {
    'Authorization': f'Bearer {token}'
}

response = requests.get(
    'https://api.example.com/api/v1/books',
    headers=headers
)

if response.status_code == 200:
    print('Books:', response.json())
elif response.status_code == 401:
    print('Token expired or invalid - please login again')
```

```java Java
String token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";

HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://api.example.com/api/v1/books"))
        .header("Authorization", "Bearer " + token)
        .GET()
        .build();

HttpResponse<String> response = client.send(request, 
    HttpResponse.BodyHandlers.ofString());

if (response.statusCode() == 200) {
    System.out.println("Books: " + response.body());
} else if (response.statusCode() == 401) {
    System.out.println("Token expired or invalid");
}
```
</CodeGroup>

<Warning>
  Always include the `Bearer` prefix before the token in the Authorization header. The format must be: `Authorization: Bearer <token>`
</Warning>

## Error Responses

### Invalid Credentials (401 Unauthorized)

Returned when username doesn't exist or password is incorrect:

```json
{
  "message": "Invalid password",
  "timestamp": "2024-03-06T10:30:00",
  "details": []
}
```

or

```json
{
  "message": "The user johndoe doesn't exist",
  "timestamp": "2024-03-06T10:30:00",
  "details": []
}
```

### Validation Error (400 Bad Request)

Returned when required fields are missing:

```json
{
  "message": "Validation failed",
  "timestamp": "2024-03-06T10:30:00",
  "details": [
    "The username is obligatory",
    "The password is obligatory"
  ]
}
```

## Complete Login Flow Example

### Full Authentication Workflow

<Steps>
  <Step title="Collect Credentials">
    Gather username and password from user input
    
    ```javascript
    const username = document.getElementById('username').value;
    const password = document.getElementById('password').value;
    ```
  </Step>
  
  <Step title="Send Login Request">
    POST credentials to the login endpoint
    
    ```javascript
    const response = await fetch('https://api.example.com/api/v1/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ username, password })
    });
    ```
  </Step>
  
  <Step title="Handle Response">
    Check response status and extract JWT token
    
    ```javascript
    if (response.status === 201) {
      const data = await response.json();
      const token = data.jwt;
      
      // Store token securely
      localStorage.setItem('authToken', token);
      localStorage.setItem('username', data.username);
      
      // Redirect to main application
      window.location.href = '/dashboard';
    } else {
      const error = await response.json();
      showError(error.message);
    }
    ```
  </Step>
  
  <Step title="Use Token in Requests">
    Include token in subsequent API calls
    
    ```javascript
    const token = localStorage.getItem('authToken');
    
    const apiResponse = await fetch('https://api.example.com/api/v1/books', {
      headers: {
        'Authorization': `Bearer ${token}`
      }
    });
    ```
  </Step>
  
  <Step title="Handle Token Expiration">
    Detect expired tokens and re-authenticate
    
    ```javascript
    if (apiResponse.status === 401) {
      // Token expired
      localStorage.removeItem('authToken');
      localStorage.removeItem('username');
      window.location.href = '/login';
    }
    ```
  </Step>
</Steps>

## Token Storage Best Practices

### Client-Side Storage Options

| Storage Method | Security | Persistence | Use Case |
|----------------|----------|-------------|----------|
| **Memory** | High | Session only | High-security applications |
| **SessionStorage** | Medium | Session only | Single-tab applications |
| **LocalStorage** | Low-Medium | Persistent | General web applications |
| **Cookies (HttpOnly)** | High | Persistent | Server-rendered applications |

### Recommendations

<Steps>
  <Step title="Choose Appropriate Storage">
    - Use memory or sessionStorage for sensitive applications
    - LocalStorage is acceptable for general-purpose apps
    - Consider HttpOnly cookies for server-side rendering
  </Step>
  
  <Step title="Implement Token Refresh">
    - Monitor token expiration time
    - Prompt user to re-authenticate before expiration
    - Clear stored token immediately upon logout
  </Step>
  
  <Step title="Secure Communication">
    - Always use HTTPS in production
    - Never log or expose tokens in console
    - Don't include tokens in URLs or query parameters
  </Step>
</Steps>

<Warning>
  Never expose JWT tokens in client-side logs, error messages, or browser console. Treat tokens as sensitive credentials.
</Warning>

## Token Expiration Handling

### Client-Side Expiration Detection

<CodeGroup>
```javascript Decode and Check Expiration
function isTokenExpired(token) {
  try {
    // Decode JWT (without verification - just reading claims)
    const payload = JSON.parse(atob(token.split('.')[1]));
    const expirationTime = payload.exp * 1000; // Convert to milliseconds
    const currentTime = Date.now();
    
    return currentTime > expirationTime;
  } catch (error) {
    return true; // Treat invalid tokens as expired
  }
}

// Usage
const token = localStorage.getItem('authToken');
if (isTokenExpired(token)) {
  // Redirect to login
  window.location.href = '/login';
}
```

```javascript Automatic Refresh Before Expiration
function scheduleTokenRefresh(token) {
  const payload = JSON.parse(atob(token.split('.')[1]));
  const expirationTime = payload.exp * 1000;
  const currentTime = Date.now();
  const timeUntilExpiry = expirationTime - currentTime;
  
  // Refresh 2 minutes before expiration
  const refreshTime = timeUntilExpiry - (2 * 60 * 1000);
  
  if (refreshTime > 0) {
    setTimeout(() => {
      // Prompt user to re-authenticate
      showReauthenticationDialog();
    }, refreshTime);
  }
}
```
</CodeGroup>

### Handling 401 Responses

```javascript
// Global axios interceptor for handling expired tokens
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response && error.response.status === 401) {
      // Clear stored token
      localStorage.removeItem('authToken');
      localStorage.removeItem('username');
      
      // Redirect to login
      window.location.href = '/login?sessionExpired=true';
    }
    return Promise.reject(error);
  }
);
```

## Password Security

### BCrypt Verification

The API uses BCrypt to securely verify passwords:

```java
// During login
if (!passwordEncoder.matches(password, userDetails.getPassword())) {
    throw new BadCredentialsException("Invalid password");
}
```

### Security Features

- **Constant-Time Comparison**: Prevents timing attacks
- **Salt Integration**: Each password has unique salt
- **Adaptive Cost**: Configurable computational complexity
- **One-Way Hash**: Cannot reverse hash to get original password

<Info>
  BCrypt automatically handles salting and verification. The `passwordEncoder.matches()` method extracts the salt from the stored hash and performs the comparison.
</Info>

## Common Issues and Solutions

| Issue | Possible Cause | Solution |
|-------|----------------|----------|
| 401 Unauthorized | Invalid username or password | Verify credentials are correct |
| 401 Unauthorized | Token expired | Re-authenticate to get new token |
| 400 Bad Request | Missing required fields | Include both username and password |
| 403 Forbidden | Account disabled | Contact administrator |
| Missing Bearer prefix | Token format incorrect | Ensure header is `Bearer <token>` |

## Security Considerations

### For API Consumers

- Store tokens securely and never expose them
- Implement proper logout to clear tokens
- Use HTTPS for all API communication
- Handle token expiration gracefully
- Never hardcode credentials in source code

### For API Developers

- Keep the JWT signing key (`security.jwt.key.private`) secure
- Use environment variables for sensitive configuration
- Implement rate limiting on login endpoint
- Log failed authentication attempts
- Consider implementing account lockout after failed attempts
