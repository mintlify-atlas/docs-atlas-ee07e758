## Overview

This page provides a comprehensive reference for all configuration properties used by the Library Management API. The main configuration file is located at:

```
src/main/resources/application.properties
```

## Complete Properties File

```properties
spring.application.name=training

# PostgreSQL Database Configuration
spring.datasource.url=${db_url}
spring.datasource.username=${db_username}
spring.datasource.password=${db_password}

# PostgreSQL Driver
spring.datasource.driver-class-name=org.postgresql.Driver

# Hibernate/JPA Configurations
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=create-drop

server.port=8081

security.jwt.user.generator=${user_jwt}
security.jwt.key.private=${key_jwt}
```

## Properties Reference

### Application Properties

<ParamField path="spring.application.name" type="string" default="training">
  The name of the Spring Boot application. This appears in logs and actuator endpoints.
  
  **Location**: `application.properties:1`
</ParamField>

### Server Properties

<ParamField path="server.port" type="integer" default="8081" required>
  The HTTP port on which the application listens for requests.
  
  **Default**: `8081`  
  **Location**: `application.properties:15`
  
  <Tip>
    Change this value if port 8081 is already in use or for different environments:
    ```properties
    # Development
    server.port=8081
    
    # Production
    server.port=8080
    ```
  </Tip>
</ParamField>

### Database Properties

<Warning>
  Database credentials must be provided via environment variables. The application will fail to start if these are not set.
</Warning>

<ParamField path="spring.datasource.url" type="string" required>
  PostgreSQL JDBC connection URL. Must be provided via the `db_url` environment variable.
  
  **Environment Variable**: `db_url`  
  **Location**: `application.properties:4`
  
  **Format**:
  ```
  jdbc:postgresql://<host>:<port>/<database_name>[?<options>]
  ```
  
  **Examples**:
  ```properties
  # Local development
  jdbc:postgresql://localhost:5432/library_db
  
  # Docker
  jdbc:postgresql://postgres:5432/library_db
  
  # Production with SSL
  jdbc:postgresql://prod-db.example.com:5432/library_db?sslmode=require
  ```
</ParamField>

<ParamField path="spring.datasource.username" type="string" required>
  Database username for authentication. Must be provided via the `db_username` environment variable.
  
  **Environment Variable**: `db_username`  
  **Location**: `application.properties:5`
  
  **Example**:
  ```bash
  export db_username="postgres"
  ```
</ParamField>

<ParamField path="spring.datasource.password" type="string" required>
  Database password for authentication. Must be provided via the `db_password` environment variable.
  
  **Environment Variable**: `db_password`  
  **Location**: `application.properties:6`
  
  <Warning>
    Never commit database passwords to version control. Use environment variables or secret management systems.
  </Warning>
</ParamField>

<ParamField path="spring.datasource.driver-class-name" type="string" default="org.postgresql.Driver">
  JDBC driver class for PostgreSQL database connectivity.
  
  **Value**: `org.postgresql.Driver`  
  **Location**: `application.properties:9`
  
  <Note>
    This driver is provided by the `postgresql` dependency in `pom.xml`.
  </Note>
</ParamField>

### JPA/Hibernate Properties

<ParamField path="spring.jpa.database-platform" type="string" default="org.hibernate.dialect.PostgreSQLDialect">
  Hibernate dialect for PostgreSQL-specific SQL generation and optimizations.
  
  **Value**: `org.hibernate.dialect.PostgreSQLDialect`  
  **Location**: `application.properties:12`
  
  This dialect enables PostgreSQL-specific features like:
  - Native UUID support
  - Array types
  - JSON/JSONB columns
  - PostgreSQL-specific functions
</ParamField>

<ParamField path="spring.jpa.hibernate.ddl-auto" type="string" default="create-drop">
  Database schema generation strategy.
  
  **Default**: `create-drop`  
  **Location**: `application.properties:13`
  
  <Warning>
    The default `create-drop` value is **only for development**. All data is lost when the application stops.
  </Warning>
  
  **Available Values**:
  
  | Value | Behavior | Use Case |
  |-------|----------|----------|
  | `create-drop` | Creates schema on startup, drops on shutdown | Development/Testing |
  | `create` | Creates schema on startup, keeps on shutdown | Development |
  | `update` | Updates schema to match entities | Development |
  | `validate` | Validates schema, no modifications | Production |
  | `none` | No schema management | Production |
  
  **Recommended Production Setting**:
  ```properties
  spring.jpa.hibernate.ddl-auto=validate
  ```
</ParamField>

### Security Properties

<Warning>
  JWT security properties must be configured via environment variables to prevent exposing secrets in version control.
</Warning>

<ParamField path="security.jwt.user.generator" type="string" required>
  JWT token issuer identifier. Used in the `iss` claim of generated tokens.
  
  **Environment Variable**: `user_jwt`  
  **Location**: `application.properties:17`  
  **Used In**: `JwtUtils.java:31`
  
  **Example**:
  ```bash
  export user_jwt="library-management-api"
  ```
  
  This value is validated when tokens are verified to ensure they were issued by this API.
</ParamField>

<ParamField path="security.jwt.key.private" type="string" required>
  Secret key used to sign and verify JWT tokens using HMAC256 algorithm.
  
  **Environment Variable**: `key_jwt`  
  **Location**: `application.properties:18`  
  **Used In**: `JwtUtils.java:34`
  
  <Warning>
    - Must be at least 256 bits (32 characters) for security
    - Must be kept secret and never committed to version control
    - Should be rotated periodically in production
  </Warning>
  
  **Generating a Secure Key**:
  ```bash
  # Generate a random 256-bit key (Linux/macOS)
  openssl rand -base64 32
  
  # Example output
  export key_jwt="7x!A%D*G-KaNdRgUkXp2s5v8y/B?E(H+"
  ```
</ParamField>

## Validation and Troubleshooting

### Verifying Configuration

Check that all required environment variables are set:

```bash
#!/bin/bash

required_vars=("db_url" "db_username" "db_password" "user_jwt" "key_jwt")

for var in "${required_vars[@]}"; do
    if [ -z "${!var}" ]; then
        echo "ERROR: $var is not set"
        exit 1
    else
        echo "✓ $var is set"
    fi
done

echo "All required environment variables are configured"
```

### Common Configuration Errors

<AccordionGroup>
  <Accordion title="Application fails to start - Missing database configuration">
    **Error**: `Error creating bean with name 'dataSource'`
    
    **Cause**: Required database environment variables not set
    
    **Solution**:
    ```bash
    export db_url="jdbc:postgresql://localhost:5432/library_db"
    export db_username="postgres"
    export db_password="yourpassword"
    ```
  </Accordion>

  <Accordion title="JWT tokens fail to validate">
    **Error**: `JWTVerificationException`
    
    **Cause**: `security.jwt.key.private` mismatch between environments
    
    **Solution**: Ensure the same JWT secret is used across all instances:
    ```bash
    # Verify the key is set
    echo $key_jwt
    ```
  </Accordion>

  <Accordion title="Port already in use">
    **Error**: `Port 8081 is already in use`
    
    **Solution**: Change the server port:
    ```properties
    server.port=8082
    ```
    Or set via environment variable:
    ```bash
    export SERVER_PORT=8082
    ```
  </Accordion>

  <Accordion title="Database schema issues in production">
    **Error**: Schema validation failures
    
    **Cause**: Using `create-drop` in production
    
    **Solution**: Change DDL auto mode:
    ```properties
    spring.jpa.hibernate.ddl-auto=validate
    ```
  </Accordion>
</AccordionGroup>

## Configuration Hierarchy

Spring Boot loads configuration in the following order (later sources override earlier ones):

1. Default properties in `application.properties`
2. Profile-specific properties (`application-{profile}.properties`)
3. Environment variables
4. Command-line arguments

**Example Override**:
```bash
# Override via environment variable
export SPRING_JPA_HIBERNATE_DDL_AUTO=validate

# Override via command line
java -jar app.jar --server.port=9090
```
